好。現在攻這個原子命題：

固定大素數 (p)。取一組 residues

[  
r_1,\dots,r_t\in\mathbb F_p^\times,  
]

其中在我們的應用中

[  
r_i=q_i^{-1}\pmod p.  
]

給定任意 shifts (\beta_i\in\mathbb R/\mathbb Z)，考慮集合

[  
A=  
\left{  
a\in\mathbb F_p:  
\left|  
\frac{a r_i}{p}+\beta_i  
\right|\le\delta  
\quad\text{for all }i=1,\dots,t  
\right}.  
]

若 (r_i) 像隨機斜率，則每個條件切掉約 (2\delta) 比例，應有

[  
|A|\approx (2\delta)^t p.  
]

取

[  
t\asymp C\log p  
]

後，若 (\delta<1/2) 固定，交集就應極小。

## 1. 先看兩個條件

對一個斜率 (r)，條件

[  
\left|  
\frac{ar}{p}+\beta  
\right|\le\delta  
]

等於

[  
ar\in I\pmod p,  
]

其中 (I\subset\mathbb F_p) 是一段長度約 (2\delta p) 的 interval。故

[  
a\in r^{-1}I.  
]

這是一個「模 (p) 的等差區間」，長度 (2\delta p)，步長 (r^{-1})。

兩個條件的交集是

[  
r_1^{-1}I_1\cap r_2^{-1}I_2.  
]

如果

[  
r_1/r_2  
]

不是很小分母的 rational approximation，這兩個模區間應近似橫截，交集約

[  
4\delta^2 p.  
]

最壞情況是 (r_1=\pm r_2)，則兩個區間方向相同，交集可達

[  
2\delta p.  
]

所以需要斜率 (r_i) 彼此不高度重合。對隨機 prime inverse residues，這應該容易。

---

## 2. Fourier 表達

令 (I_i) 是長度 (2\delta p) 的 interval。其 Fourier 展開為

# [  
1_{I_i}(x)

\sum_{\xi\in\mathbb F_p}  
\widehat{1_{I_i}}(\xi)  
e_p(\xi x),  
]

其中

[  
e_p(u)=e^{2\pi i u/p}.  
]

交集大小：

# [  
|A|

\sum_{a\in\mathbb F_p}  
\prod_{i=1}^t 1_{I_i}(ar_i).  
]

展開後：

# [  
|A|

p  
\sum_{\xi_1r_1+\cdots+\xi_t r_t=0}  
\prod_{i=1}^t  
\widehat{1_{I_i}}(\xi_i).  
]

主項是所有 (\xi_i=0)：

[  
p(2\delta)^t.  
]

其他項由線性關係

[  
\xi_1r_1+\cdots+\xi_t r_t=0  
]

控制。若 (r_i) 沒有小係數線性關係，則非主項小。

因 interval Fourier 係數滿足

[  
|\widehat{1_I}(\xi)|  
\le  
\min(2\delta,, C/|\xi|_p),  
]

其中 (|\xi|_p) 是到 (0) 的最小代表大小。大係數項自然小；真正危險的是小 (\xi_i) 的線性關係。

所以我們需要：

[  
\boxed{  
r_1,\dots,r_t\text{ 對小係數線性關係 dissociated。}  
}  
]

---

## 3. 隨機斜率的 dissociation

若 (r_i) 是在 (\mathbb F_p^\times) 中隨機選取，則對任何非零小係數向量

[  
(\xi_1,\dots,\xi_t),  
]

方程

[  
\sum_i \xi_i r_i=0  
]

成立概率約 (1/p)。

小係數向量數量若限制

[  
|\xi_i|\le H  
]

則約

[  
(2H+1)^t.  
]

若取

[  
t=C\log p,  
]

這可能很大；但可先取 (H) 固定或多項式很小。需要平衡。

我們不必證完全無小關係；只需交集小。可選 (t) 不太大，例如

[  
t=c\log p  
]

且用高階 Fourier/large sieve 控制。這是可行但技術。

## 4. 也許可以用更簡單的隨機覆蓋論證

若 (r_i,\beta_i) 都隨機，則對固定 (a)，每個條件獨立，概率 ((2\delta)^t)。但 (\beta_i) 在我們問題中是 adversarial，不能隨機。

然而若 (r_i) 隨機，對任意兩個不同 (a,a')，事件

[  
a r_i+\beta_i\in I_i,  
\qquad  
a' r_i+\beta_i\in I_i  
]

要求

[  
(a-a')r_i  
]

落入差集 (I_i-I_i)，大小約 (4\delta p)。所以概率約 (4\delta). Thus pairs correlation decays.

可以用二階矩：

Let

[  
N=|A|.  
]

Then

[  
\mathbb E N=(2\delta)^t p.  
]

And

[  
\mathbb E N^2  
\le  
\mathbb E N+  
p^2(4\delta)^t.  
]

如果 (t=C\log p), and choose δ small so (p^2(4δ)^t) small, then (N) small with high probability.

But we need uniform over shifts (\beta_i). The pair estimate may be uniform over shifts because difference condition only depends on (I_i-I_i). Good.

More precisely, for random independent (r_i), for fixed (a\ne a'), the probability over (r_i) that both conditions hold is at most (C\delta) per i, uniformly in shifts. Thus

[  
\mathbb E N^2\le p(2\delta)^t+p^2(C\delta)^t.  
]

If (C\delta<1), taking

[  
t>A\log p  
]

makes this (<1) or small. Then with high probability (N) is small for fixed shifts.

But shifts are continuous/discrete many. Here shifts (\beta_i) come from (b_{q_i}p^{-1}/q_i); for each (q_i), (\beta_i) has (q_i) possible values. The total number of shift choices is

[  
\prod_i q_i,  
]

huge, about (Y^t). Cannot union bound naively if (t\log Y\sim(\log p)^2), maybe possible if probability is super-exponentially small? Need check.

Second moment gives probability N≥1 ≤ E N, not enough for all shifts.

Need a uniform deterministic bound over shifts, not probabilistic shifts.

Fourier/dissociation may give uniform over shifts more naturally.

## 5. VC dimension / discrepancy possibility

The family of sets

[  
{a: ar_i/p+\beta_i\in I}  
]

as (\beta_i) varies are intervals in the cyclic order induced by (r_i). We need intersection of one interval from each of t random cyclic orders. There are only (p) possible endpoints per i, so number of distinct sets per i is (O(p^2))? Intervals of fixed length have (p) shifts. Total shift choices (p^t), large but maybe random orders make all intersections small with high probability by known results on random permutations: intersection of intervals in random permutations.

Indeed multiplication by random (r_i) is a random cyclic permutation of (\mathbb F_p) restricted to group automorphisms, not fully random but close enough.

For fully random permutations, t intervals of density δ have intersection size about δ^t p uniformly over intervals once t≈C log p. This is standard via VC/discrepancy/union bound over p^t intervals? Number choices p^t, probability of large intersection ~exp(-c δ^t p). If δ^t p maybe p^{-A}, not enough. Need choose t so expected intersection maybe log p; uniform bound.

If δ^t p ≈ C log(p^t)=C t log p, then intersection O(t log p). For t~c log p, need δ^t p≈(log p)^2. This gives t≈(log p)/|\log δ|, feasible. Uniform intersection size polylog, not 1. That may be enough for entropy compression.

This is plausible.

## 6. We only need weak intersection bound

We don't need intersection ≤1. It suffices to reduce each vertex entropy from (\log p) to, say, (0.1\log p). That is, intersection size ≤p^{0.1}.

For this, choose t constant large enough? Since random density δ^t, need

[  
δ^t p\le p^{0.1},  
]

i.e.

[  
t\ge 0.9\frac{\log p}{|\log δ|}.  
]

So still (C\log p), but uniform union easier if allow p^{0.1}.

Number of shift choices ~p^t=exp(C(log p)^2). Probability of bad intersection for random permutations can be exp(-c p^{0.1}), which beats it. So uniform high probability possible.

Good.

## 7. Deterministic existence of test slopes

Thus for each p, there exists a test set (q_1,\dots,q_t) such that for all shifts, intersection size ≤p^{0.1}. We need choose these q's from actual neighboring prime blocks and simultaneously for many p. Random selection plus union over p likely works, since failure probability super-small.

This avoids Kloosterman? It treats q_i mod p as random residues. To justify for primes q_i, we need distribution of primes modulo p. Since q_i ranges over intervals compatible with p, length may be large enough (Irving range gives q up to p^{3/2}, down to p^{2/3}). For q interval longer than p maybe good; if q<p, distribution modulo p over primes in shorter interval is harder. But we can choose neighbor block with q>p, say (\ell\approx 3k/2), so interval length (p^{3/2}), much larger than p. Then primes mod p are well-distributed by PNT in AP/Bombieri-Vinogradov? Modulus p and length p^{3/2}; should be okay even by classical estimates on average, perhaps enough. Irving also covers.

So choose test neighbors from upper-scale block (Q\sim p^{3/2}). Then q mod p is well-distributed. Great.

This is an important design choice: for entropy tests, use larger neighbor scales.

## 8. Consequence for global graph

For each block (P_k), choose test neighbors from block (P_{\ell}) with (\ell\approx 3k/2). This is within Irving range boundary. It provides random slopes modulo p with enough length. Then each vertex's residue entropy is compressed if many test edges pass.

Since graph includes these edges, large Fourier coefficient forces pass on most of them. Thus entropy compression works.

This is quite promising.

## 9. How to fold into Fourier sum

The entropy argument would show:

For any threshold R, the number of characters (\lambda) with energy ≤R is at most

[  
\exp(o(\log L))\cdot e^{C R}  
]

or more concretely much less than (e^{\log L}). Need enough so

[  
\sum_\lambda e^{-cQ(\lambda)}  
]

is small.

If each vertex entropy reduces by factor p^{0.1}, total low-energy character count ≤  
[  
\exp(0.1\log L)  
]  
which is still enormous compared to (1/\sigma). Need energy decay (e^{-cR}) with R maybe proportional to number of failed edges. For very low energy R, count p^{0.1 total} too large.

Need stronger compression: low-energy should force residues to a global h, not just p^{0.1} choices each. Otherwise count remains (L^{0.1}), far too large.

But maybe iterating tests across graph layers reduces entropy to global dimension. Agreement propagation can do that. The intersection bound per vertex alone not enough.

Need combine: once each vertex has small candidate list, edge constraints between vertices enforce consistency among lists, reducing to global h. This is list-agreement.

Still plausible but more complex.

## 10. Alternative: choose t so intersection ≤1 for each vertex

If each vertex's residue determined by test neighbors, then total choices are residues on test neighbor sets. But those neighbor sets overlap; recursively, a small seed may determine all vertices. If choose a global seed set of O(log L?) vertices, count may be manageable.

Could use expander property: choose a small seed layer; constraints propagate to determine next layer, etc. Number of seed choices maybe product of primes in seed, should be ≤e^{o(log L)}. Need small seed relative to all vertices. Since graph expands, possible.

This is essentially agreement testing again.
