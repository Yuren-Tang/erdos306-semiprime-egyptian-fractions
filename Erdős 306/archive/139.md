好。現在設計 **bounded-overlap block skeleton**。目的有三個：

[  
\boxed{  
\text{連通傳播 label；有足夠 boundary；能量不重複計算。}  
}  
]

如果直接用 full compatible graph，block 邊很多，boundary 很強，但同一批 prime-edge 可能在多個論證中被反覆使用，bookkeeping 複雜。更穩的方法是先抽一個稀疏骨架。

---

# 1. 尺度骨架的基本選擇

令 block index 為

[  
k_0,k_0+1,\dots,K.  
]

我們不必只連相鄰 (k,k+1)，因為 Irving 最有效的尺度比例大約是

[  
X_{\ell}=X_k^{4/3}.  
]

若

[  
X_k=2^k,  
]

則

[  
\ell\approx \frac43 k.  
]

所以定義一條尺度鏈：

[  
k_{j+1}=\left\lfloor \frac43 k_j\right\rfloor.  
]

對應 blocks：

[  
P_{k_j},P_{k_{j+1}},P_{k_{j+2}},\dots  
]

在這條鏈上放完整二部 block edges：

[  
P_{k_j}\times P_{k_{j+1}}.  
]

這條鏈 Irving-compatible，且 label 可沿鏈傳播。

但單鏈 boundary 太弱，可能又出現 isolated interval 問題。因此需要多條 offset chains。

---

# 2. 多 offset chains

取若干初始點

[  
k_0+s,\qquad 0\le s<S,  
]

其中 (S) 可以取

[  
S\asymp k_0  
]

或較小的常數倍窗口，使每個尺度 (k) 至少落入若干條鏈的鄰近位置。

每條鏈：

[  
k^{(s)}_{j+1}=\left\lfloor \frac43 k^{(s)}_j\right\rfloor.  
]

把所有鏈的相鄰 block-pairs 加入 skeleton。

這樣得到 block skeleton (\mathcal S)。

它的性質：

1. 每條 skeleton edge 對應一個完整二部 prime graph，尺度比約 (4/3)，適合 Irving。
    
2. 每個 block 出現在 (O(1)) 或 (O(\log K)) 條 skeleton edges 中，取 offset 可控制 bounded overlap。
    
3. 尺度圖整體連通，而且任何非大 component 會切斷若干 skeleton edges，形成 boundary。
    

---

# 3. Bounded overlap

我們需要：

[  
\sum_{(k,\ell)\in\mathcal S}  
Q_{k\ell}(a)  
\le  
C Q_{\mathrm{full}}(a).  
]

這若 (\mathcal S\subset E_{\mathrm{full}})，且每個 prime edge只出現一次，甚至有

[  
\sum_{\mathcal S}Q_{k\ell}\le Q_{\mathrm{full}}.  
]

因每個 skeleton block edge 對應 disjoint 的 prime-edge set

[  
P_k\times P_\ell,  
]

不同 block pairs 不重疊，所以 bounded overlap 沒問題。只要不在證明中重複使用同一 block pair 的能量即可。

因此 skeleton 的能量可直接由 full energy 支配。

---

# 4. Boundary expansion

需要證：若 (C) 是 skeleton block graph 中的 proper component，則 boundary edge數不少，或者至少 boundary edge 的 weighted cost 足以支付 (C) 的 entropy。

這裏有兩種層次。

### block-level entropy

選擇 component 的 block indices，熵約

[  
O(|\partial C|\log K)  
]

對一個 bounded-degree expander skeleton。若 skeleton 只是幾條鏈，interval component boundary 只有 (O(1))，但 block shape entropy也不大。不過 component 內 prime residue entropy巨大，不能只靠 shape。

### vertex/phase entropy

若 component (C) 自己有 independent ordinary label (m_C)，其 label entropy約

[  
\log |m_C|  
]

不大。

若 component 內 residues arbitrary，則必須由內部 skeleton edges產生 local labels；否則 internal energy支付。也就是 component 不會有 raw entropy without internal energy。

所以 boundary expansion只需支付「獨立 component label」和 boundary exceptions；內部自由度由遞歸 good-component lemma處理。

這是合理的。

---

# 5. 為何單鏈不夠，offset chains有何用？

單鏈上，一個長 interval component只需兩條 boundary edges。若這個 interval有一個 independent label (m')，boundary cost只有兩個 block pair。  
但 label (m') 的 entropy只有

[  
\log |m'|,  
]

不是整個 interval體積，因為內部低能要求整個 interval服從同一 label。這可能已夠。

若 interval內不是同一 label，內部 good-component/bad-boundary會再分解並付費。

所以單鏈也許足夠做 Peierls。問題是 mass / variance 可能需要所有 dyadic scales，而單鏈太稀疏，(\sum 1/(pq)) mass只有幾何級？  
鏈尺度 (k_{j+1}=4k_j/3) 時 reciprocal mass per link roughly

[  
\frac{1}{k_j k_{j+1}},  
]

sum over geometrically growing (k_j) converges, not enough for adjustable (1/b). Full construction需要 many scales / edges. But for minor arc control，skeleton不必承載全部 mass；只要能控制 characters。主弧方差用 full graph。

所以 skeleton可以稀疏，只用於控制；full graph提供更多 mass和能量。

單鏈是否控制所有 primes? 若只含鏈上的 blocks，其他 blocks未受控。故需要 offset chains覆蓋所有 blocks。

---

# 6. 一個更簡單的 skeleton：nearest-scale graph

也可取 bounded-degree graph：

每個 block (k) 連到

[  
\ell_+(k)=\lfloor 4k/3\rfloor  
]

和

[  
\ell_-(k)=\lfloor 3k/4\rfloor  
]

若在範圍內。

這樣每個 block度數 (O(1))，圖大致連通。  
但是否真的連通？映射 (k\mapsto \lfloor4k/3\rfloor) 可能按乘法軌道分成多個 components。可再加入少量 local edges (k\leftrightarrow k+1) if compatible. Adjacent dyadic scales ratio (2) between (2^k) and (2^{k+1}) may exceed Irving upper (3/2) in prime size? Actually (X_{k+1}=2X_k), ratio 2, outside (x^{3/2})? For modulus (q\sim2X), variable (p\sim X), (p=q/2), within (q^{2/3}\le p\le q^{3/2}) for large q. So adjacent dyadic blocks are Irving-compatible too. Good.

Then simply use edges (k\leftrightarrow k+1)!  
Scale ratio 2 in prime size is certainly within (q^{2/3}) to (q^{3/2}). If (p\sim X), (q\sim2X), then (p\asymp q), fine.

Why earlier used (4/3)? It gave a convenient power saving point, but Irving range allows comparable scales. So skeleton can be nearest-neighbour chain (k,k+1).

This is much simpler.

---

# 7. Nearest-neighbour skeleton

Define

[  
\mathcal S={(k,k+1):k_0\le k<K}.  
]

Each skeleton edge is complete bipartite

[  
P_k\times P_{k+1}.  
]

Properties:

1. Connected chain covering all blocks.
    
2. Bounded overlap trivial.
    
3. Boundary of interval component has at most two edges; sufficient for label entropy, while internal disorder is recursively handled.
    
4. Irving applies because scales comparable.
    
5. Dense agreement visible condition: for (P_k,P_{k+1}), (X\sim Y), visible cutoff up to (B\lesssim X). Major label size (M\sim1/\sigma\sim k_0X_0) may exceed lowest (X_0) by (k_0), so lowest pairs cannot see full major arc. But for label propagation we can use higher blocks where (X_k>M). For low blocks, once label established at higher scale, Irving correction pulls downward.
    

This reintroduces anchor but on a chain. However good-component lemma can choose seed at higher visible segment.

---

# 8. Covering major label size

Major labels satisfy

[  
|m|\le C/\sigma\asymp k_0X_0.  
]

Thus need blocks with primes

[  
X_k\gg k_0X_0  
]

to see all major labels. This is only

[  
k-k_0\asymp \log_2 k_0  
]

higher. Very modest.

So choose skeleton starting from (k_0) up to (K), and define a **visibility zone**

[  
k\ge k_{\mathrm{vis}}=k_0+\lceil \log_2 k_0\rceil+C.  
]

Above this, all major arc labels are visible.  
Good component in visibility zone gives seed (m).  
Then label propagates downward to lower blocks via Irving correction.

This is promising.

For larger ordinary labels (m) beyond major arc, they belong to higher energy layers; to see them need higher visibility zones. But counting bound allows (m) up to (\sqrt R/\sigma). For energy layer (R), need blocks with

[  
X_k\gg \sqrt R/\sigma.  
]

So visibility zone depends on (R). If (R) huge, need high (k). But perhaps skeleton extends to K enough for relevant R; top issue again. However now if (m) huge, Gaussian energy large; summation may tolerate. Still counting needs visibility up to (\sqrt R/\sigma).

This is manageable by splitting R up to top visibility, and high R via maybe stronger energy. But not fully solved.

---

# 9. Bounded-degree chain and no giant component

On nearest-neighbour chain, good components are intervals. If no giant good interval in visibility zone, there are many bad edges or intervals. Boundary bad edges count comparable to number of components, not length. Could a long bad interval have huge entropy but only two bad boundaries? Inside interval edges are bad or not good; their internal bad cost accumulates along length. Good.

Thus chain skeleton may suffice for combinatorial structure.

Counting recursively along chain resembles one-dimensional transfer matrix. This may be far easier than high-dimensional Peierls.

---

# 10. Transfer-matrix viewpoint

Nearest-neighbour skeleton yields a Markov chain / transfer operator:

# [  
Z_{\mathcal S}

\sum_{a_{P_{k_0}},\dots,a_{P_K}}  
\prod_{k}  
e^{-cQ_{k,k+1}(a_k,a_{k+1})}.  
]

If we can show this transfer operator has effective rank (\asymp1/\sigma) dominated by ordinary labels, done. This is an operator/Poincaré problem.

Local agreement says the kernel

[  
K_k(a_k,a_{k+1})=e^{-cQ_{k,k+1}}  
]

is large mainly when both blocks share an ordinary label visible at scale k. Thus the transfer operator contracts all non-label directions. This is perhaps the cleanest formulation.

This resembles one-dimensional spin chain with huge spins but strong ferromagnetic kernel. Transfer matrix methods could work.

---

# 11. New possible route

Use nearest-neighbour chain and prove inductively:

Let (F_k(a_k)) be partial partition function up to block k. Show

[  
F_k(a_k)  
\le  
\sum_m W_k(m)\mathbf 1[a_k\approx m]  
+  
\text{small error},  
]

where (W_k(m)) is Gaussian weight.  
The kernel to next block propagates labels and suppresses nonmatching residues by Irving/dense agreement.

This avoids complicated graph components. It is a dynamic programming / transfer theorem.

This may be more tractable than arbitrary block graph Peierls.

---

# 12. Potential issue

A single transition (P_k\to P_{k+1}) may not fully suppress all entropy, especially for labels larger than visible scale. But as k grows, visibility increases. Transfer induction can carry a distribution over labels (m) with size up to current scale; larger unresolved modes are damped or postponed. This is multiscale but sequential, not arbitrary tower.

This seems promising.

---

# 13. Next step

We should formulate **chain transfer Poincaré lemma**:

For skeleton chain (P_{k_0},\dots,P_K),

# [  
Z_{\mathcal S}

\sum_{a}e^{-c\sum_k Q_{k,k+1}(a)}  
\ll  
\frac1{\sigma_{\mathcal S}}.  
]

If skeleton partition bound holds, full graph bound follows since

[  
Q_{\mathrm{full}}\ge Q_{\mathcal S}  
]

so wait:

[  
e^{-Q_{\mathrm{full}}\le e^{-Q_{\mathcal S}},  
]

therefore

[  
Z_{\mathrm{full}}\le Z_{\mathcal S}.  
]

Thus controlling skeleton suffices.

Great. We can discard full graph complexity and focus on a chain.

This is important.
