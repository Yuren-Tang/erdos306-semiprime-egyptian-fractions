我們先對剛才的 **pairwise CRT compatibility lemma** 做壓力測試。這一步很重要：如果它是假的，整個「統一 inverse theorem」就只是把困難包裝起來。

我們有一組素數頂點 (P)，每個頂點 (p) 有 residue

[  
a_p\in\mathbb F_p.  
]

對邊 (pq)，令 (H_{pq}) 是由 CRT 決定的整數 residue：

[  
H_{pq}\equiv a_p\pmod p,\qquad  
H_{pq}\equiv a_q\pmod q,\qquad 0\le H_{pq}<pq.  
]

邊相位就是

[  
\frac{H_{pq}}{pq}\pmod1.  
]

若 Fourier 係數大，很多邊要滿足

[  
\left|\frac{H_{pq}}{pq}\right|  
]

很小，也就是 (H_{pq}) 很接近 (0) 或 (pq)。為簡潔，先看接近 (0) 的情形：

[  
H_{pq}\le \delta pq.  
]

這意味著 (a_p,a_q) 能由一個小於 (\delta pq) 的整數同時表示。

我們想推出：存在一個小整數 (h)，使

[  
a_p\equiv h\pmod p  
]

對多數 (p) 成立。

---

## 1. 這個命題可能為假嗎？

構造一個潛在反例。令每個頂點 (p) 指派一個「局部小數」：

[  
a_p\approx \alpha p  
]

其中 (0<\alpha<\delta)。也就是 (a_p) 是約 (\alpha p) 的 residue。

對兩個素數 (p,q)，CRT glue (H_{pq}) 一般不會約為 (\alpha pq) 或小數；CRT 解通常在 ([0,pq)) 中像隨機分佈。除非這些 residues 來自同一個整數 (h)。例如若

[  
a_p\equiv h\pmod p  
]

且 (h<\delta pq)，則 (H_{pq}=h)。這是主弧。

所以「比例型」不是反例。

再試：把頂點分成兩群 (P_1,P_2)，在 (P_1) 上 residues 來自小整數 (h_1)，在 (P_2) 上來自小整數 (h_2)。則群內邊相位小，群間邊相位一般不小。若圖不連通或有弱 cut，這會形成反例。但我們的 dyadic block graph 應該有 expansion；若群間邊很多，會被殺。

這說明 inverse lemma 需要 graph expansion，不只是 pairwise local condition。

再試更微妙的：讓 (h_p) 隨頂點變化，但相鄰頂點 (p,q) 的 (h_p,h_q) 很接近，且都小於 (\delta pq)。如果

[  
a_p\equiv h_p\pmod p,  
\qquad  
a_q\equiv h_q\pmod q,  
]

那 CRT glue 小不意味 (h_p=h_q)；它要求存在一個 (H_{pq}) 同時等於 (h_p) mod (p) 和 (h_q) mod (q)，且 (H_{pq}) 小。若 (h_p,h_q) 本身都小於 (p,q)， then (H_{pq}) is not generally close to either unless (h_p=h_q). In fact if (H_{pq}<\min(p,q)), then (H_{pq}=h_p=h_q). More generally if (H_{pq}<\delta pq) but may exceed (p,q), there may be more flexibility.

So small edge phase does not force equality of local (h_p), but it imposes strong CRT compatibility.

---

## 2. Exact small window: what does it imply?

Suppose for an edge (pq),

[  
H_{pq}\le \delta pq.  
]

For fixed (p) and (a_p), what (a_q)'s are allowed?

The possible (H)'s are

[  
H=a_p+mp,\qquad 0\le m<q.  
]

Condition (H\le \delta pq) allows

[  
0\le m\le \delta q - a_p/p.  
]

So there are about

[  
\delta q  
]

possible residues (a_q=H\bmod q).

Thus a single edge condition only restricts (a_q) to a (\delta)-fraction of residues. Not enough.

But if (q) is connected to many (p)'s with different (a_p,p), the allowed sets intersect. For many random constraints, intersection shrinks to about (\delta^{d}q), forcing few possibilities. This is the expansion mechanism.

Thus the inverse lemma is plausible but must be probabilistic/entropy-based, not deterministic pair equality.

---

## 3. Entropy proof outline

Let (A_p) be the random variable/residue choice at vertex (p). A large Fourier coefficient corresponds to many edge constraints:

[  
H_{pq}\in I_{pq},  
]

where (I_{pq}) is a short interval near (0) or (pq).

For a fixed vertex (q), each neighbor (p) imposes a condition on (a_q) of density about (2\delta) in (\mathbb F_q). If these conditions were independent, (d) neighbors would reduce possibilities to

[  
(2\delta)^d q.  
]

Once (d\gg\log q), (a_q) is essentially forced.

Our graph degrees are much larger than (\log q). Therefore low energy should force residues at most vertices to be determined by a small global parameter. That is exactly what we need.

This is now a **constraint satisfaction / entropy compression** problem.

---

## 4. But dependencies matter

The neighbor constraints depend on (a_p). If the (a_p)'s are adversarially chosen, the allowed sets for (a_q) might align and fail to shrink. Alignment itself means the (a_p)'s are compatible with a common global (h). So this is again an inverse phenomenon.

For example, if all (a_p\equiv h\pmod p) with small (h), then all constraints align and (a_q=h) survives. This is the intended major arc.

So a robust theorem should say:

Either entropy compresses strongly, or residues are globally aligned to a small (h).

This is exactly the right dichotomy.

---

## 5. This resembles agreement testing

There is a known combinatorial pattern: if local pairwise data are consistent on many edges of an expander graph, then there is a global object agreeing with most vertices. In theoretical CS this appears as agreement tests / direct product tests.

Here local data are residues (a_p), and an edge passes if their CRT glue lies in a short interval. Passing many edges should imply agreement with some global integer (h) on many vertices.

This is a useful conceptual import. It suggests the inverse lemma is not absurd; it is a kind of CRT-valued agreement theorem on an expander.

---

## 6. A clean possible theorem

Let (G=(P,E)) be a sufficiently dense expander-like graph. Let each (p\in P) carry (a_p\in\mathbb F_p). Define edge pass if

[  
\left|  
\frac{H_{pq}}{pq}  
\right|\le \delta,  
]

where (H_{pq}) is the CRT glue of (a_p,a_q).

Then:

**CRT agreement theorem, rough form.**  
If at least (1-\varepsilon) proportion of edges pass, then there exists an integer (h) with

[  
|h|\le C\delta M  
]

for an appropriate scale (M), such that

[  
a_p\equiv h\pmod p  
]

for all but (O(\varepsilon+\delta)) proportion of vertices.

The exact bound on (h) must be scale-aware, because edge moduli (pq) vary. In dyadic blocks, one first proves it block-wise, then glues blocks through scale graph.

This would unify:

- small diagonal major arcs: global (h) small;
    
- non-diagonal characters: fail agreement, hence many bad edges, hence decay;
    
- high diagonal characters: if (h) is not small, agreement exists but edge pass fails at scales where (pq\ll h) or through wrap-around.
    

---

## 7. Is the theorem actually true with interval near both 0 and 1?

Edge phase small means

[  
H_{pq}\in [0,\delta pq]\cup[(1-\delta)pq,pq).  
]

The second case corresponds to a global integer (h) near (-)small modulo (pq). So major arcs include both small positive and small negative (h), i.e.

[  
|h| \le \delta pq  
]

in symmetric representatives.

Thus no issue: use symmetric CRT representative

[  
H_{pq}^{*}\in(-pq/2,pq/2],  
]

and pass if

[  
|H_{pq}^{*}|\le\delta pq.  
]

Then global (h) is small in symmetric sense.

## 8. How to prove agreement theorem locally?

For three vertices (p,q,r). Suppose edges (pq) and (pr) pass with small representatives (H_{pq},H_{pr}). They both reduce to (a_p\mod p). Difference

[  
H_{pq}-H_{pr}  
]

is divisible by (p), and has size at most

[  
\delta pq+\delta pr\asymp \delta pQ  
]

if (q,r\asymp Q). Thus

[  
H_{pq}\equiv H_{pr}\pmod p,  
]

but many possible multiples of (p) remain, about (\delta Q). Need more edges, not just triangles.

If also edge (qr) passes, then (H_{pq},H_{pr},H_{qr}) are pairwise compatible modulo p,q,r. One may show there exists a single (H_{pqr}) modulo pqr whose pair projections are small; this may force all three pair representatives close to one common small integer if (\delta) small enough? ## 9. Alternative entropy proof might be simpler than constructive agreement

For our Fourier sum, we don't need explicitly recover h; we just need count low-energy characters. Entropy compression can be done without naming h.

Consider exposing residues (a_p) vertex by vertex. For each new vertex q, it has many neighbors among previously exposed vertices. Passing conditions restrict (a_q) to intersection of many sets each of density (O(\delta)). Unless these sets align. If they align, that means previous residues already have a common compatible h, reducing entropy globally.

This is complex but plausible.

This is a real research-level combinatorial inverse theorem.

## 10. Should we be worried?

Yes, this is the point where a lot of hidden difficulty could live. Your suspicion is right. We have replaced case splitting with a beautiful inverse theorem, but that theorem is not trivial. It may be as hard as the original if mishandled.

However, it is a **sharper and more standard-looking** problem: a CRT agreement/inverse theorem on an expander-like prime graph. That is progress: it isolates the remaining combinatorial core.

---

## 11. Is there a way to avoid proving such a heavy inverse theorem?

Maybe yes: use a **sequential construction** instead of global random Fourier.

Return to absorption but now with mass handled by choosing blocks in central windows. Sequential methods avoid summing over all high frequencies (h) globally. They require only local surjectivity at each step, which Irving gives.

This is the tradeoff:

- Global Fourier: elegant, but needs hard high-frequency inverse theorem.
    
- Sequential absorption: messy layers, but avoids global aperiodicity and local CLT at modulus (L).
    

Earlier absorption failed because of mass separation. But with block windows and planned remainder, maybe sequential can handle mass without global high-frequency.

Maybe that route is actually easier.

---

## 12. Re-evaluate route choice

Global Fourier now hits a serious inverse theorem. Sequential route hits bookkeeping but perhaps more controllable.

A hybrid may be best:

1. Use deterministic/greedy mass planning to ensure each block target lies in central window.
    
2. For each block, use finite-field Fourier only to solve local residues, not full (L)-scale exact equality.
    
3. Exact mass achieved by choosing block targets from a prearranged Egyptian fraction identity at block scale.
    

But then one must construct exact block target masses; back to Egyptian fractions.

Global Fourier was meant to avoid that.

So no free lunch.

## 13. Current honest status

The strongest route remains global Fourier + Irving + local CLT, but its hard core is now:

[  
\boxed{\text{CRT agreement / aperiodicity inverse theorem for large Fourier coefficients.}}  
]

This is nontrivial. It may be provable with existing agreement-testing / inverse Littlewood-Offord ideas, but not immediate.

This is a reasonable point to pause and say:

- We found a plausible high-level proof architecture.
    
- Irving handles prime reciprocal distribution.
    
- Mass issue forces global or carefully coupled treatment.
    
- The remaining new combinatorial problem is not solved yet.
    

But since you ask to continue, next useful step is to formulate the CRT agreement theorem more precisely in a dyadic block setting and see whether a simple version can be proved for **complete bipartite graph between two blocks**. If even that fails, global route is in trouble. If it works, extend by block graph.
