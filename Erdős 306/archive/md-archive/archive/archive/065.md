好。現在改成較自然的 **平均版 list-decoding**。這一步的目的，是避免上一版過強的要求：

[  
\text{對所有 shifts，所有頂點都 list 很小。}  
]

那太硬，可能需要很重的 discrepancy / sum-product 工具。其實 Fourier 求和不需要這麼強；它只需要證：

[  
\boxed{  
\text{能讓很多邊同時相位小的 assignments 總數很少。}  
}  
]

也就是直接數低能量 assignments，而不是先對每個頂點做最壞情形 list bound。

---

## 1. 重新定義低能量集合

令 (P) 是頂點素數集合，(G=(P,E)) 是 dyadic prime block graph。對 assignment

[  
a=(a_p)_{p\in P},\qquad a_p\in\mathbb F_p,  
]

定義邊相位

# [  
\phi_{pq}(a)

\frac{a_p q^{-1}}p+  
\frac{a_q p^{-1}}q  
\quad\in\mathbb R/\mathbb Z.  
]

定義能量

[  
Q(a)=  
\sum_{pq\in E}|\phi_{pq}(a)|^2.  
]

Fourier 係數滿足

[  
|\widehat\mu(a)|  
\le  
\exp(-cQ(a)).  
]

所以我們要控制

[  
\sum_{a\notin\mathfrak M}e^{-cQ(a)}.  
]

等價地，按能量層分解：

# [  
\mathcal A_R

{a: Q(a)\le R}.  
]

若能證

[  
|\mathcal A_R|  
\le  
|\mathfrak M_R|\cdot e^{C R}  
]

其中 (\mathfrak M_R) 是主弧附近的低維族，那麼 Fourier 求和可閉合。

---

## 2. 用隨機測試邊壓低 (|\mathcal A_R|)

從全圖 (E) 中隨機抽一個測試子圖 (T)，例如每個頂點抽

[  
t=C\log p  
]

條上層測試邊。

若 (Q(a)\le R)，則大多數邊相位小；因此在隨機測試子圖中，期望上也有大多數測試邊相位小。由 Markov/Chernoff，對多數低能量 (a)，測試子圖能看到它的低能量性。

也就是：

[  
a\in\mathcal A_R  
\Longrightarrow  
a\text{ 通過多數測試邊}  
]

除去一個可由 (R) 支付的例外。

然後我們只需要數「通過測試子圖」的 assignments。

這是平均版 list-decoding：不是對任意 shifts 做 uniform bound，而是把整個 assignment 放進測試圖一起數。

---

## 3. 測試圖上的約束數量

假設測試圖 (T) 有 (M_T) 條邊。每條測試邊要求

[  
|\phi_{pq}(a)|\le\delta  
]

或在能量版本中要求大多數邊如此。

對隨機 assignment (a)，一條邊通過概率約為

[  
2\delta.  
]

若測試邊近似獨立，通過 ((1-\rho)M_T) 條邊的概率約為

[  
\exp(-cM_T).  
]

總 assignments 數是

[  
L=\prod_{p\in P}p.  
]

所以粗略估計：

[  
#{\text{通過測試者}}  
\approx  
L e^{-cM_T}.  
]

要把它壓到主弧規模，需要

[  
M_T\gg \log L.  
]

而如果每個頂點抽 (C\log p) 條測試邊，則

# [  
M_T  
\asymp  
\sum_{p\in P}\log p

\log L.  
]

正好夠！

這是一個非常好的信號。它說：每頂點 (O(\log p)) 條測試邊，總測試約束數正好在 entropy 尺度上。

---

## 4. 為什麼這比單點 list-decoding 好？

單點 list-decoding 只說：

[  
a_p\text{ 從 }p\text{ 種降到 }p^\varepsilon\text{ 種}.  
]

全局 entropy 仍是

[  
\varepsilon\log L.  
]

但平均測試圖直接對全 assignment 計數；若測試邊近似獨立，就可以得到

[  
L e^{-cM_T}  
]

而 (M_T\asymp\log L)，所以可以壓到

[  
L^{1-cC}.  
]

取 (C) 大，這能壓過任意 (L^\eta)，甚至接近常數級。

換言之，**不必先產生每個頂點的小 list；直接數通過測試圖的 assignments 更有效。**

這是重要簡化。

---

## 5. 依賴問題

當然，測試邊不完全獨立。邊共享頂點，且 assignment 可結構化。

但我們可以選測試圖為近似有向森林／低交疊圖：

- 每個頂點 (p) 選上層鄰居 (q);
    
- 讓尺度嚴格上升，例如 (k\to\lfloor3k/2\rfloor);
    
- 這樣測試邊形成一個有向 acyclic scale graph。
    

若尺度單調上升，約束可以按層暴露。給定上層 residues，下層每個 (a_p) 受到 (C\log p) 個近獨立模區間限制。這回到單點 list-decoding，但在全局計數中使用，不做最壞 shifts 的 union。

具體地，從最高層往下數：

1. 最高層 residues 任意；
    
2. 下一層每個頂點由上層鄰居 shifts 限制；
    
3. 每個 (a_p) 可選數平均約 (p e^{-ct});
    
4. 乘起來得到全局 entropy 壓縮。
    

這比 uniform-over-shifts 更自然，因為 shifts 已經固定為上層 residues，而我們在整體計數中一起求和。

---

## 6. 層向上測試圖

構造：

對每個 dyadic block (P_k)，選測試邊到 block

[  
P_{\lceil 3k/2\rceil}.  
]

對每個

[  
p\in P_k  
]

選

[  
t_k=Ck  
]

個鄰居

[  
q\in P_{\lceil3k/2\rceil}.  
]

因為

[  
\log p\asymp k.  
]

尺度嚴格上升，所以沒有 directed cycles。最高若超過 (K)，可停止或接到 reservoir layer。

這個測試圖用於 entropy bound；候選圖本身仍可包含更多邊以提供質量與 Fourier 衰減。

---

## 7. 自頂向下計數

設最高層 residues 任意。對一個下層頂點 (p)，其測試鄰居 (q_1,\dots,q_t) 的 residues 已經固定，於是 shifts (\beta_i) 固定。

我們需要估計有多少

[  
a_p\in\mathbb F_p  
]

使得至少 ((1-\rho)t) 條測試邊通過。

這正是 list size

[  
L_p(\beta).  
]

但現在不需要對所有可能 (\beta) 單獨 union bound；在計數中，我們對上層 residues 求和。若平均 list size 滿足

[  
\mathbb E_{\beta}|L_p(\beta)|  
\le p e^{-ct},  
]

則總計數會壓縮。

上層 residues (\beta_i) 不是獨立均勻於 (\mathbb R/\mathbb Z)，但如果上層 assignments 尚未受太強結構，或用 induction 加權計數，可以處理。這裏可能需要 martingale/entropy 方法。

---

## 8. 平均 list size actually easy

對固定斜率 (r_i)，如果 shifts (\beta_i) 是均勻隨機，則對任何 (a_p)，每條測試通過概率 (2\delta)，獨立。因此

[  
\mathbb E_{\beta}|L_p(\beta)|  
\le  
p e^{-ct}.  
]

在全圖計數中，上層 residues (a_{q_i}) 可能近似均勻 when summing over all assignments. Since (a_q) ranges over (\mathbb F_q), the shift

[  
\beta_i=\frac{a_qp^{-1}}q  
]

is uniform over q-grid in (\mathbb R/\mathbb Z), sufficiently close.

Therefore average list size over upper residues is indeed

[  
\ll p e^{-ct}.  
]

This is much easier than uniform shifts.

This is a major simplification.

---

## 9. Entropy compression estimate

For each lower vertex (p), choose

[  
t=C\log p.  
]

Then

# [  
p e^{-ct}

# p e^{-cC\log p}

p^{1-cC}.  
]

Take (C) so large that

[  
1-cC<-!A  
]

or at least (<\varepsilon). Then each vertex contributes a factor (p^\varepsilon) or less on average. Across many layers, total entropy collapses.

In fact, if (C) is very large, average choices per lower vertex can be (<1), meaning only globally coherent assignments survive.

This is exactly what we need.

---

## 10. 這裏的代價

測試圖使用了大量邊：

[  
\sum_p C\log p\asymp C\log L.  
]

這仍遠少於候選圖所有邊，不影響質量設計。它只是用於證明 large Fourier coefficients rare。

但測試邊必須是候選邊的一部分，且在 Irving-compatible range。選 (k\to3k/2) 正好可行。

---

## 11. 這使 agreement propagation 變簡單了嗎？

某種程度上，是。原先需要：

[  
\text{local lists} + \text{global agreement propagation}.  
]

現在用有向 acyclic testing 可以直接做 entropy compression，可能不必單獨證完整 agreement theorem。

最高層 residues仍可能任意，形成 seed entropy。若最高層總 (\log)-mass 太大，還不夠。解法：

- 讓 testing graph continue upward until a very high cap layer;
    
- cap layer size can be chosen small or handled by主弧/diagonal?
    
- 或雙向測試，讓沒有自由頂層。
    

如果尺度一直 (k\to1.5k)，在有限 (K) 會到頂。頂層 entropy約

[  
\sum_{p\in P_K}\log p\asymp 2^K,  
]

可能接近 total (\log L)，不能忽略.

So need avoid leaving a huge free top layer.

Options:

1. Use cyclic scale testing: also edges from high layers down to lower layers, creating agreement cycles. Then no free top, but counting harder.
    
2. Choose top layer small relative to total entropy. Since total over k0..K is dominated by largest k anyway ((\sum 2^k)), top layer is not small. Bad.
    
3. Use many scale bands not exponentially growing? Still top dominates in count.
    

Thus acyclic counting alone leaves too much entropy. Need either downward tests too or global agreement propagation.

This is important.

---

## 12. Use two-sided tests

For each block (k), choose tests both upward to (1.5k) and downward to (2k/3). Then no layer is free; graph has cycles. Counting cannot be simple top-down, but entropy method on expanders may handle. This returns to agreement testing.

However, we can choose a small seed set cutting across scales and show all other vertices determined by tests; but due to cycles, possible.

Thus propagation remains necessary, but local average list-decoding gives the ingredient.

---

## 13. Current improved picture

We no longer need worst-case uniform list-decoding. We need:

1. Local average list-decoding: easy via averaging over neighbor residues.
    
2. Expansion/agreement propagation to remove free entropy.
    

Agreement testing tools are precisely for step 2.

This is a cleaner and more plausible path.
