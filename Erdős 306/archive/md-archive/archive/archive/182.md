這裏最誠實的做法不是再假裝它已經完了，而是立刻把那個縫隙精確命名，然後攻它。

# 真正剩下的 single-block 局部問題

經 base vertex (p_0) 後，每個 vertex (p) 有 list

[  
L_p={n\in[-B,B]: n\equiv a_p\pmod p},  
]

而 base list 是

[  
\mathcal L=L_{p_0}.  
]

大多數 (p) 滿足

[  
L_p\cap\mathcal L\ne\varnothing.  
]

我們需要證：

[  
\boxed{  
\text{若 good-edge graph 很稠密，則某個 }m\in\mathcal L  
\text{ 屬於大多數 }L_p.  
}  
]

也就是存在 dominant label：

[  
m\in L_p  
\quad\text{for most }p.  
]

先前我用「把每個 (p) 任意指派到一個 (m\in L_p\cap\mathcal L)」來處理，然後用 cross-label energy 壓多 classes。這一步在直覺上對，但任意指派會丟失「一個 vertex 可能同時支持多個 labels」的信息，導致很多小 classes 時估計退化。正確處理應直接對 lists 做 agreement，而不是先任意分配。

---

# 改正方向：AP-list agreement lemma

我們應把 single-block 的核心引理改寫成：

**AP-list agreement lemma.**  
令

[  
P\subset[X,2X],  
]

對每個 (p\in P) 給出

[  
L_p={n\in[-B,B]:n\equiv a_p\pmod p}.  
]

若 intersection graph

[  
p\sim q  
\quad\Longleftrightarrow\quad  
L_p\cap L_q\ne\varnothing  
]

密度至少 (1-\eta)，則存在

[  
m\in[-B,B]  
]

使

[  
m\in L_p  
]

對 (1-O(\eta^c)) 比例的 (p) 成立；或者沒有此 (m) 時，bad edges / cross energy 足以支付熵。

這才是 single-block theorem 的正確局部核心。

---

# 為什麼這比任意 class bookkeeping 更好？

因為它保留了 list 的交叉信息。  
任意指派

[  
p\mapsto m(p)  
]

會把

[  
L_p\cap L_q\ne\varnothing  
]

這個事實丟掉。兩個 vertex 雖然被指派到不同 labels，仍可能通過第三個 label 相交；這正是先前 cross-energy bookkeeping 退化的原因。

如果直接處理 lists，稠密 pairwise intersections 應該迫使一個 global label 出現。這是一種 agreement theorem，而不是普通 Peierls bookkeeping。

---

# 下一步應做什麼？

現在最正確的下一步是證這個 **AP-list agreement lemma**，或至少證它在我們需要的範圍

[  
B\asymp X\log X\sqrt R  
]

內成立。

證明思路如下：

1. 選 regular base vertex (p_0)，令  
    [  
    \mathcal L=L_{p_0}.  
    ]
    
2. 對大多數 (p)，  
    [  
    L_p\cap\mathcal L\ne\varnothing.  
    ]
    
3. 假設沒有 (m\in\mathcal L) 覆蓋大多數 (p)。則 base list 中的覆蓋分散。
    
4. 因 good graph 稠密，許多 (p,q) 仍滿足  
    [  
    L_p\cap L_q\ne\varnothing.  
    ]
    
5. 對三角形 (p_0,p,q)，若  
    [  
    L_{p_0}\cap L_p,\quad  
    L_{p_0}\cap L_q,\quad  
    L_p\cap L_q  
    ]  
    都非空，但沒有共同 label，則這是一個「local inconsistency triangle」。
    
6. 需要證：大量 local inconsistency triangles 會造成大量能量，或者在 arithmetic progression lists 裏不可能大量存在。
    

這是現在真正的局部題。

現在我們應該繼續攻：

\boxed{\text{AP-list agreement lemma}}
